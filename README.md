# Football Video Analysis Pipeline

## Overview

This pipeline analyzes football match videos to:

1. **Assign teams** to each player (`home`, `away`, `gk`)
2. **Detect the referee** (not included in original ground truth)
3. **Determine jersey colour names** for each team

The original ground truth files (`gt.txt`) only contain player bounding boxes with track IDs `1–22`. Referees, coaches, ball boys, and sideline staff are NOT included.

Because of this:

- Team identity must be inferred from jersey appearance
- Referees must be detected separately

# Feature 8 — Team Assignment

## Goal

Assign every tracked player to:

- `home`
- `away`
- `gk` (goalkeeper/outlier)

using jersey colour similarity.

---

# Why This Is Needed

The provided MOT-style ground truth only gives:

- frame number
- track ID
- bounding box coordinates

It does NOT specify:

- team identity
- jersey colour
- player role

Therefore team membership must be estimated visually.

---

# Pipeline

## Step 1 — Jersey Region Extraction

Instead of using the full player bounding box, only the torso region is extracted.

This avoids:

- grass pixels
- shorts/socks
- heads/hair
- crowd/background noise

Multiple vertical torso bands are tested, and the most colour-consistent region is selected.

---

## Step 2 — Jersey Colour Signature

The selected jersey crop is converted to HSV colour space.

A compact histogram descriptor is created:

- 16 hue bins
- 4 saturation bins
- 4 value bins

Total:

```text
16 + 4 + 4 = 24-dimensional feature vector
```

This acts as the jersey’s colour fingerprint.

---

## Step 3 — Team Clustering

For each frame:

- all player descriptors are collected
- K-Means clustering (`k=2`) separates players into two major groups

These clusters represent:

- home team
- away team

Players far from both cluster centers are marked as:

```text
gk
```

This usually captures:

- goalkeepers
- visually inconsistent players
- noisy detections

---

## Step 4 — Temporal Stabilization

Raw clustering can swap labels between frames.

Example:

```text
Frame 1:
Cluster A = home

Frame 2:
Cluster A = away
```

To prevent this:

### Centroid Anchoring

Cluster centroids are matched over time so team labels stay consistent.

---

## Step 5 — GK Rescue Logic

Some normal players occasionally become outliers due to:

- motion blur
- lighting
- occlusion
- pose variation

A temporal voting system rescues these players.

### Rescue Rule

If a track receives:

```text
>30% home/away labels
```

across all frames, it is reassigned from `gk`.

---

# Why 30% Works

Empirical observations:

| Player Type | Typical Team Assignment Rate |
|---|---|
| Normal field players | 45–55% |
| Goalkeepers / rare outliers | <15% |

So 30% provides a reliable separation threshold.

# Feature 9 — Jersey Colour Naming

## Goal

Determine the dominant jersey colour name for each team using the crops generated by Feature 8.

---

# Important Clarification

The issue is **NOT specifically orange kits**.

The real issue is:

> Some jerseys are occasionally classified as incorrect colours due to colour ambiguity, lighting variation, shadows, compression artifacts, or small accent regions.

In the current test set, many of the incorrect predictions happened to be labelled as `"orange"` by chance, but orange itself is not the root problem.

The system can misclassify any colour under difficult visual conditions.

---

# Why the Center-Pixel Method Was Chosen

The approach intentionally uses the center region because it:

- is extremely fast
- avoids most background contamination
- works well for solid-colour football kits
- performs reliably in most sequences

---

# Pipeline

## Step 1 — Collect Center Pixels

For every jersey crop:

- the center pixel (or small center region) is sampled
- pixels are grouped by predicted team

---

## Step 2 — Remove Outliers

Outlier pixels are removed using standard deviation filtering.

Pixels farther than:

```text
2 standard deviations
```

from the mean are discarded.

This removes:

- shadows
- highlights
- noisy crops
- motion blur artifacts

---

## Step 3 — Compute Dominant Colour

The remaining pixels are averaged to obtain a stable dominant colour.

---

## Step 4 — Convert HSV to Colour Name

The dominant HSV value is mapped to a human-readable colour label.

---

# HSV Colour Mapping

| Hue Range | Colour |
|---|---|
| 0–10, 165–179 | Red |
| 11–25 | Orange |
| 26–34 | Yellow |
| 35–85 | Green |
| 86–99 | Cyan |
| 100–130 | Blue |
| 131–150 | Purple |
| 151–164 | Pink |
| Low saturation + high value | White |
| Low value | Black |

---

# Example Output

```text
[v_HdiyOtliFiw_c003]

[home]
4759 crops
3296 inliers
HSV=(26,147,151) → yellow

[away]
6277 crops
4632 inliers
HSV=(124,27,197) → white

home : yellow
away : white
```

---

# Current Limitation

Misclassifications may occur because of:

- lighting changes
- compression noise
- shadows
- motion blur
- thin stripes or accents
- saturated broadcast effects
- ambiguous HSV boundaries

The current approach prioritizes:

```text
speed + simplicity + robustness
```

over fine-grained colour precision.

Despite occasional mistakes, it performs reliably on the majority of sequences.

# Referee Detection

# Problem

The original `gt.txt` contains ONLY player annotations.

Referees are completely absent.

YOLO detects all people in the scene, including:

- players
- referees
- linesmen
- coaches
- ball boys
- spectators near sidelines

The challenge is identifying which detection corresponds to the referee.

---

# Detection Strategy

## Step 1 — Find Unmatched YOLO Detections

Each YOLO person detection is compared against all GT player boxes using IoU.

If:

```text
IoU < 0.5
```

against every GT player box, the detection becomes a referee candidate.

---

## Step 2 — Distance-to-Players Metric

For each unmatched candidate:

- compute center point
- measure Euclidean distance to every GT player center
- compute average distance

Formula:

```text
avg_distance =
average(
distance(candidate_center, player_center)
)
```

---

## Step 3 — Select Closest Candidate

The candidate with the smallest average distance is selected.

Why?

Because referees typically move:

```text
inside the active player cluster
```

rather than near the sidelines.

---

## Step 4 — Vertical Validation

The vertical player range is computed using GT boxes.

If the selected candidate lies:

- above all players
- below all players

it is rejected.

This removes:

- linesmen
- sideline staff
- ball boys
- spectators

---

# Why This Works

## Referee Behavior

Referees:

- follow play closely
- stay near player formations
- move centrally

---

## Non-Referee Behavior

Linesmen and staff typically remain:

- near boundaries
- outside player clusters
- vertically separated from gameplay

The distance-based method naturally exploits this difference.

# Referee GT Format

Same MOT format as original GT:

```text
frame,track_id,x,y,w,h,confidence,-1,-1,-1
1,0,545,191,76,187,127.34,-1,-1,-1
```

Notes:

- `track_id = 0` always represents referee
- `confidence = average distance to players`
- lower confidence value = stronger referee candidate
